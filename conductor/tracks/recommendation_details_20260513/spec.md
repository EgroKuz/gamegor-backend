# Specification: Recommendations Details Page

## Overview
This track implements a dedicated Recommendation Details page for a specific gaming session. When a user submits a review or views an existing session, they can navigate to this page to see the AI-generated advice associated with that specific session. The page will display the game title, the AI advice, associated tags, and provide a direct link to watch videos related to that specific game.

## Functional Requirements
- **Route:** Create a new route, e.g., `/recommendations/:sessionId` or `/sessions/:sessionId/recommendation`, to display the details for a specific session's recommendation.
- **Data Fetching:** Fetch the specific session/recommendation data from the backend API using the ID from the URL.
- **UI Elements:** The page must display:
    - **Game Title:** The name of the game associated with the session.
    - **AI Advice:** The text generated by the AI advisor.
    - **Tags:** The tags associated with the session/recommendation.
    - **Navigation Button:** A button labeled "Watch Videos" (or similar).
- **Navigation Action:** Clicking the "Watch Videos" button must navigate the user to the `VideoContentPage`, specifically filtered for the game associated with this recommendation (e.g., passing the game title or ID via URL parameters or state).

## Non-Functional Requirements
- **UX/UI:** The design should match the application's dark, neon-accented aesthetic. The AI advice should be prominently displayed and easily readable.
- **Loading/Error States:** The page must handle and display appropriate loading spinners and error messages if the API request fails.

## Acceptance Criteria
- Navigating to the recommendation route for a valid session ID displays the game title, AI advice, and tags.
- Clicking the "Watch Videos" button correctly routes the user to the video page with the context of the current game.
- The page renders correctly on both desktop and mobile viewports.

## Out of Scope
- Modifying the AI generation logic on the backend.
- Creating a general list view of all recommendations (this track focuses on the single, specific recommendation view per session).
